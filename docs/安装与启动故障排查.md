# 安装与启动故障排查（Windows / PowerShell）

适用项目：`ai_tarot_saas`  
适用场景：`pip install fastapi uvicorn openai python-dotenv` 报错、前后端启动失败、联调失败。

启动步骤总览请看：`docs/启动命令.md`

## 1. 推荐的稳定安装流程（先做这个）

```powershell
cd "C:\Users\Zhisen Pei\Desktop\ai_tarot_saas"

# 首次执行：创建虚拟环境
python -m venv .venv

# 激活虚拟环境
.\.venv\Scripts\Activate.ps1

# 若提示脚本策略限制（按需执行）
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass

# 升级安装工具
python -m pip install --upgrade pip setuptools wheel

# 安装后端依赖
python -m pip install fastapi uvicorn openai python-dotenv

# 验证是否安装成功
python -m pip show fastapi uvicorn openai python-dotenv
```

关键点：始终用 `python -m pip`，不要直接用 `pip`，避免装到错误的 Python 环境。

## 2. pip 安装报错排查

### 问题 A：`'pip' 不是内部或外部命令`

原因：`pip` 未在 PATH，或当前终端没激活正确环境。  
解决：

```powershell
python -m pip --version
python -m pip install fastapi uvicorn openai python-dotenv
```

### 问题 B：`No module named pip`

解决：

```powershell
python -m ensurepip --upgrade
python -m pip install --upgrade pip
```

### 问题 C：权限错误（`Access is denied` / `WinError 5`）

原因：写入系统目录被拒绝。  
解决：使用项目本地虚拟环境，不要装到系统 Python。

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install fastapi uvicorn openai python-dotenv
```

### 问题 D：网络/SSL 超时（`ReadTimeout`、`SSL CERTIFICATE_VERIFY_FAILED`）

解决：切换源并添加 trusted host。

```powershell
python -m pip install fastapi uvicorn openai python-dotenv -i https://pypi.org/simple --trusted-host pypi.org --trusted-host files.pythonhosted.org
```

### 问题 E：安装成功但运行仍报 `ModuleNotFoundError`

原因：安装和运行用的不是同一个 Python。  
排查：

```powershell
where python
python --version
python -m pip --version
```

确认 `python -m pip --version` 输出路径和你当前解释器一致。

## 3. 后端启动报错排查

当前后端启动命令：

```powershell
cd "C:\Users\Zhisen Pei\Desktop\ai_tarot_saas\backend"
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 问题 A：`Set-Location ... positional parameter cannot be found`

原因：路径中有空格（`Zhisen Pei`），但没加引号。  
解决：`cd` 路径必须加双引号。

### 问题 B：`Address already in use` / 8000 被占用

```powershell
Get-NetTCPConnection -LocalPort 8000 -ErrorAction SilentlyContinue |
Select-Object -ExpandProperty OwningProcess -Unique |
ForEach-Object { Stop-Process -Id $_ -Force }
```

### 问题 C：`Error loading ASGI app. Could not import module "main"`

原因：目录不对。  
解决：必须先 `cd` 到 `backend` 再执行 `uvicorn main:app`。

### 问题 D：`/draw_card` 返回 `请先配置 ALIYUN_API_KEY`

原因：`backend/main.py` 里仍是占位符。  
解决：把 `ALIYUN_API_KEY = "YOUR_API_KEY_HERE"` 改成真实 Key 后重启后端。

## 4. 前端启动报错排查

前端启动命令：

```powershell
cd "C:\Users\Zhisen Pei\Desktop\ai_tarot_saas\frontend"
npm install
npm run dev -- -p 3000
```

### 问题 A：`Port 3000 is in use`

```powershell
Get-NetTCPConnection -LocalPort 3000 -ErrorAction SilentlyContinue |
Select-Object -ExpandProperty OwningProcess -Unique |
ForEach-Object { Stop-Process -Id $_ -Force }
```

### 问题 B：`Unable to acquire lock at frontend/.next/dev/lock`

```powershell
# 先结束前端相关 node 进程
Get-CimInstance Win32_Process |
Where-Object { $_.Name -eq 'node.exe' -and $_.CommandLine -like '*ai_tarot_saas\\frontend*' } |
Select-Object -ExpandProperty ProcessId -Unique |
ForEach-Object { Stop-Process -Id $_ -Force }

# 再删 lock
Remove-Item "C:\Users\Zhisen Pei\Desktop\ai_tarot_saas\frontend\.next\dev\lock" -Force -ErrorAction SilentlyContinue
```

### 问题 C：`npm` 命令不可用

先确认 Node 安装：

```powershell
node --version
npm --version
```

若失败，重新安装 Node.js LTS（安装后重开终端）。

## 5. 前后端联调自检（启动后）

### 1) 检查后端

```powershell
Invoke-RestMethod -Uri http://127.0.0.1:8000/
```

期望返回：

```json
{"message":"Welcome to AI Tarot API"}
```

### 2) 检查占卜接口

```powershell
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:8000/draw_card -ContentType "application/json" -Body '{"question":"我是否该在近期换工作？"}'
```

期望返回包含：`card` 和 `meaning`。

### 3) 检查前端

浏览器打开：`http://127.0.0.1:3000`

## 6. 最常见的根因总结

1. 没激活 `.venv`，导致包装错环境。  
2. 路径有空格但 `cd` 没加引号。  
3. 3000/8000 端口被残留进程占用。  
4. Next.js 残留 `.next/dev/lock`。  
5. 阿里云 API Key 仍是占位符。  

按本文件第 1 节流程重做一次，通常可一次性恢复。
